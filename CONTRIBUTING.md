# Contributing to Skills Manager

Skills should be shareable. So should the tools that manage them. Contributions of all kinds are welcome — bug reports, feature ideas, documentation fixes, and code.

## Prerequisites

- Node.js ≥ 18
- npm ≥ 9

## Local setup

```bash
git clone https://github.com/talktoajayprakash/skillsmanager.git
cd skillsmanager
npm install
npm run build
npm link   # makes `skillsmanager` point to your local build
```

After `npm link`, the `skillsmanager` command in your shell runs your local source.

## Development workflow

```bash
npm run build       # compile TypeScript
npm test            # run tests with vitest
```

The source lives in `src/`. The compiled output goes to `dist/`. Never commit `dist/` — it is generated by CI.

## Submitting issues

- Search existing issues before opening a new one
- For bugs, include your OS, Node version (`node -v`), and the full error output
- For feature requests, describe the use case, not just the solution

## Submitting pull requests

1. Fork the repo and create a branch: `git checkout -b fix/your-topic`
2. Make your changes with tests where applicable
3. Run `npm run build && npm test` to verify everything passes
4. Open a PR against `main` with a clear description of what and why

**Guidelines:**
- Keep PRs focused — one concern per PR
- Do not add new runtime dependencies without prior discussion in an issue
- Follow the existing TypeScript style (strict mode, no `any`)
- Update docs if your change affects CLI behavior or the registry format

## Releasing

Releases are tag-based and handled by maintainers. If you think a release is overdue, open an issue.

## Questions?

Open a [GitHub Discussion](https://github.com/talktoajayprakash/skillsmanager/discussions) — not an issue — for general questions.
